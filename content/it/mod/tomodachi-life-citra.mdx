<Section icon={Sparkles} title="Impostazioni Ottimali dell'Emulatore per Grafica e Audio">

Ottenere un frame rate fluido e un audio chiaro nel gioco richiede regolazioni all'interno dei pannelli di configurazione dell'emulatore. Le impostazioni predefinite sono progettate per la massima compatibilità su hardware di fascia bassa, il che spesso si traduce in modelli di personaggi pixelati e un output audio confuso.

All'interno della scheda di configurazione grafica, i giocatori possono regolare le impostazioni per migliorare la fedeltà visiva. La regolazione della risoluzione interna trasforma il display pixelato a 240p della console originale in un'esperienza nitida in alta definizione.

| Nome Impostazione | Valore Predefinito | Valore Consigliato | Impatto |
|--------------|---------------|-------------------|--------|
| Risoluzione Interna | 1x Nativa | 3x o 4x Nativa | Riduce drasticamente i bordi seghettati |
| Filtro Texture | Nessuno | Anime o Bicubico | Leviga gli elementi dell'interfaccia |
| Layout Schermo | Predefinito | Affiancato | Visualizza chiaramente entrambi gli schermi |
| Tipo di Shader | Shader Normale | Shader Separato | Previene cali di frame durante le transizioni |
| Stretching Audio | Abilitato | Disabilitato | Riduce il ritardo e il crepitio dell'audio |

<Callout type="tip" title="Ottimizzazione del Layout dello Schermo">
L'uso del layout dello schermo affiancato facilita la visualizzazione degli eventi principali dell'isola, tenendo d'occhio gli aggiornamenti di stato dei Mii sul secondo schermo touch.
</Callout>

La configurazione audio è altrettanto importante per questo titolo, poiché il gioco si affida pesantemente alla sintesi vocale text-to-speech. Se si verifica uno stuttering nell'emulazione audio, le voci sintetizzate risulteranno distorte, rendendo difficili da capire i dialoghi. Disabilitare lo stretching audio e selezionare il backend audio corretto di solito stabilizza l'output vocale.

<Section icon={Target} title="Installazione Passo-Passo e Importazione del Gioco">

La configurazione del gioco richiede il posizionamento dei file nelle directory corrette e la configurazione dell'emulatore affinché riconosca la tua libreria di giochi. Segui questi passaggi per preparare il tuo ambiente.

<Steps>
  <Step num="1" title="Acquisire e Installare l'Emulatore">
    Scarica l'ultimo installer stabile per il tuo sistema operativo. Esegui il file eseguibile e segui le istruzioni a schermo per completare l'installazione.
  </Step>
  <Step num="2" title="Preparare la ROM Decriptata">
    Assicurati che il file del gioco sia in un formato decriptato. L'emulatore non può leggere direttamente i file criptati dalle cartucce fisiche senza una corretta conversione.
  </Step>
  <Step num="3" title="Crea una Directory di Gioco Dedicata">
    Crea una cartella sull'unità di archiviazione denominata "Games". Sposta il file del gioco decriptato in questa cartella per mantenere organizzata la tua directory.
  </Step>
  <Step num="4" title="Configura il Percorso della Directory">
    Apri l'emulatore, fai doppio clic sulla finestra principale e seleziona la cartella che hai creato. L'icona del gioco dovrebbe apparire nell'elenco della libreria principale.
  </Step>
</Steps>

<Callout type="warning" title="Formati di File">
Assicurati che il file del gioco utilizzi l'estensione corretta. L'emulatore supporta principalmente file decriptati e gli archivi compressi devono essere decompressi prima dell'avvio.
</Callout>

Una volta impostata la directory del gioco, verifica che il gioco si avvii correttamente. Se il software non si carica, controlla la console dei log per identificare componenti di sistema mancanti o errori di formato.

</Section>

<Section icon={Shield} title="Risoluzione dei Glitch Comuni e Correzioni">

I giocatori che eseguono il gioco su un emulatore potrebbero riscontrare bug visivi o audio specifici che non esistevano sull'hardware originale. Il problema più comune è il glitch dei volti Mii mancanti, in cui i personaggi appaiono con texture facciali vuote o corrotte.

Questo problema si verifica perché l'emulatore non ha accesso ai font di sistema proprietari utilizzati dal sistema operativo originale per il rendering dei tratti del viso. La risoluzione richiede l'importazione dei file di sistema condivisi nella directory dell'emulatore.

| Descrizione del Problema | Causa Principale | Soluzione Consigliata |
|-------------------|---------------|----------------------|
| Volti Mii Vuoti | Font di sistema mancanti | Importa i file dei font di sistema nella cartella user |
| Schermo Nero all'Avvio | File di gioco criptato | Decripta il file del gioco utilizzando strumenti esterni |
| Crepitio Audio | Desincronizzazione buffer | Cambia il backend audio in cubeb o SDL2 |
| Salvataggio Corrotto | Chiusura impropria | Salva sempre nel gioco e attendi la fine della scrittura |
| Frame Rate Bassi | Scaling risoluzione eccessivo | Abbassa la risoluzione interna a 2x o 3x |

<Callout type="success" title="Risoluzione Volti Mii">
Dopo aver copiato i file dei font condivisi nella directory di sistema, riavvia l'emulatore. Le modifiche verranno applicate immediatamente, ripristinando tutti i tratti del viso ai Mii esistenti e nuovi.
</Callout>

Se i cali di prestazioni persistono durante il gameplay, disabilitare gli shader hardware o svuotare la cache degli shader può risolvere lo stuttering causato dalla compilazione di nuove risorse visive al volo.

</Section>
