<Section icon={Sparkles} title="성능을 위한 최적의 설정">

모바일 기기에서 최신 시스템을 에뮬레이션하려면 그래픽 파이프라인과 CPU 에뮬레이션 모드를 세심하게 튜닝해야 합니다. 잘못된 구성을 사용하면 낮은 프레임 속도, 시각적 아티팩트 또는 발열 스로틀링이 발생할 수 있습니다. 아래에 설명된 구성은 시각적 충실도와 열 효율의 균형을 맞추도록 최적화되었습니다.

| 카테고리 | 설정 항목 | 권장 값 | 성능 영향 |
| :--- | :--- | :--- | :--- |
| 그래픽 | 그래픽 API | Vulkan | 높음 (개선) |
| 그래픽 | 해상도 배율 | 1x 또는 0.75x | 중간 (안정성) |
| 그래픽 | 셰이더 캐시 | 활성화 (Enabled) | 끊김 현상 감소 |
| 그래픽 | VSync | 비활성화 (Disabled) | 프레임 속도 향상 |
| 시스템 | CPU 정확도 | 보통 (Normal) | 높은 성능 |
| 시스템 | 멀티코어 에뮬레이션 | 활성화 (Enabled) | 높은 코어 활용도 |

모바일 GPU에서는 Vulkan을 OpenGL보다 우선적으로 선택하는 것이 중요합니다. Vulkan은 하드웨어에 직접 액세스하여 CPU 오버헤드를 줄여주기 때문입니다. Snapdragon 칩셋 사용자는 커스텀 Turnip 드라이버를 임포트하여 렌더링 파이프라인을 더욱 최적화할 수 있습니다. 중급형 기기에서는 장시간 플레이 시 발열 스로틀링을 방지하기 위해 해상도 배율을 0.75x로 낮추는 것을 강력히 권장합니다.

<Callout type="success" title="성능 향상 팁">
비동기 셰이더(Asynchronous Shaders)를 활성화하면 새로운 아파트에 들어가거나 섬 지도를 처음 로드할 때 발생하는 미세한 끊김 현상을 크게 줄일 수 있습니다.
</Callout>

</Section>

<Section icon={Swords} title="일반적인 충돌 및 버그 문제 해결">

토모다치 라이프: 리빙 더 드림은 고유한 시스템 요구 사항을 가진 복잡한 생활 시뮬레이션 게임이므로, 플레이어들은 **Eden 에뮬레이터**를 사용할 때 특정 렌더링 및 입력 버그를 자주 겪게 됩니다. 원활한 플레이를 위해 이러한 문제를 우회하는 방법을 이해하는 것이 필수적입니다.

| 문제 설명 | 근본 원인 | 해결 방법 | 효과 |
| :--- | :--- | :--- | :--- |
| 시작 시 충돌 | 오래된 셰이더 캐시 또는 파이프라인 | 셰이더 캐시 및 파이프라인 저장소 삭제 | 매우 효과적 |
| 키보드 프리징 | 소프트웨어 키보드 애플릿 충돌 | 애플릿 설정을 'Real Applet'으로 변경 | 매우 효과적 |
| 검은 잔디 글리치 | 텍스처 매핑 호환성 문제 | 섬 지형을 모래 지형으로 사용 | 보통 |
| 낮은 FPS / 렉 | 발열 스로틀링 또는 높은 해상도 | 해상도 배율을 0.75x로 감소 | 매우 효과적 |

키보드 프리징 버그는 게임이 플레이어에게 Mii의 이름이나 발음을 입력하도록 요청할 때 발생합니다. 소프트웨어 키보드 애플릿이 로드되지 않으면 게임이 무기한 중단됩니다. 이를 해결하려면 에뮬레이터의 애플릿 설정에 액세스하여 키보드 애플릿 구성을 'Real Applet'으로 변경하십시오. 키보드가 여전히 표시되지 않으면 취소 또는 뒤로 버튼을 빠르게 누르면 시스템 입력이 새로 고침될 수 있습니다.

검은 잔디 글리치는 섬의 잔디 지역이 단색의 검은색 텍스처로 렌더링되는 광범위한 렌더링 문제입니다. 공식 에뮬레이터 패치를 기다리는 동안 실용적인 해결 방법은 섬의 기본 지면 유형을 모래로 변경하는 것입니다. 모래는 올바르게 렌더링되어 시각적 버그를 피할 수 있습니다.

<Callout type="warning" title="데이터 손실 방지">
텍스트 입력 상자와 상호작용하기 전에 항상 게임 내 메뉴를 통해 수동으로 게임 진행 상황을 저장하십시오. 자동 저장이 실행되지 않은 상태에서 예상치 못한 키보드 충돌이 발생하면 진행 상황을 잃을 수 있습니다.
</Callout>
