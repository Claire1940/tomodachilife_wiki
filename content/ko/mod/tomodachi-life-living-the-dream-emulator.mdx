## 최적의 성능을 위한 에뮬레이터 설정

파일 준비가 끝났다면, 실제 플레이를 위해 게임 디렉터리를 정확히 연결해야 합니다. 게임 목록이 비어 보인다면 설정 메뉴로 이동해 ROM 파일이 들어 있는 폴더를 수동으로 추가하세요.

### 단계별 초기 설정

1. **키 설치:** 에뮬레이터의 Actions 메뉴를 열고 "Install Keys"를 선택해 `prod.keys` 파일을 불러오세요.
2. **펌웨어 설정:** "Install Firmware" 옵션을 사용해 시스템 환경이 게임 요구 사항과 일치하도록 맞추세요.
3. **그래픽 설정:** Graphics 탭으로 이동해 백엔드를 **Vulkan**으로 전환하세요. 이렇게 하면 잔디가 회색으로 보이거나 배경 요소가 사라지는 일반적인 문제를 방지할 수 있습니다.
4. **디렉터리 매핑:** Options > Settings로 이동해 게임 파일이 저장된 디렉터리를 추가하세요.

### 커스텀 텍스처 관리

커스텀 에셋을 사용하려면 먼저 페이스 페인트가 적용된 Mii처럼 최소 하나의 커스텀 요소가 포함된 기존 세이브 파일이 있어야 합니다. 그래야 도구가 게임 데이터와 상호작용하는 데 필요한 디렉터리 구조가 생성됩니다.

| 기능 | 역할 | 사용 팁 |
| :--- | :--- | :--- |
| **User Save Dir** | 세이브 데이터 위치 찾기 | 편집 전에 항상 파일을 백업하세요 |
| **Refresh** | 에셋 동기화 | 파일을 디렉터리로 옮긴 후 사용하세요 |
| **Replace** | 텍스처 교체 | 먼저 백업 폴더를 만들어 두세요 |

> **💡 팁:** 반드시 "UGC Backups" 폴더를 만들어 두세요. 게임 텍스처를 교체하기 전에 소프트웨어에서 백업 경로를 선택하라고 요구합니다. 이렇게 하면 커스텀 텍스처 로드 실패 시 원본 게임 에셋이 영구적으로 손실되는 일을 막을 수 있습니다.

기본 게임의 공식 업데이트와 커뮤니티 토론은 [Nintendo Support page](https://www.nintendo.com/)에서 확인할 수 있습니다.

## FAQ

**Q: tomodachi life living the dream emulator에서 커스텀 텍스처를 사용해도 안전한가요?**

A: 네, 커뮤니티에서 검증된 신뢰할 수 있는 도구를 사용하고 원본 세이브 파일 백업을 항상 유지한다면 안전합니다. 별도의 폴더에 안전 복사본 없이 파일을 덮어쓰지 마세요.

**Q: 왜 게임 화면이 "회색"이거나 물 빠진 것처럼 보이나요?**

A: 대부분 에뮬레이터를 OpenGL 백엔드로 실행할 때 발생합니다. 에뮬레이터 설정에서 그래픽 백엔드를 Vulkan으로 바꾸면 정상적인 텍스처 렌더링이 복원됩니다.

**Q: 원래 콘솔 세이브를 에뮬레이터로 옮길 수 있나요?**

A: 네, 일반적으로 실제 하드웨어에서 세이브 파일을 내보낸 뒤 에뮬레이터의 사용자 세이브 디렉터리에 넣으면 됩니다. 사용하는 에뮬레이터 버전이 요구하는 형식과 파일 구조가 일치하는지 확인하세요.

**Q: 실행하려면 고사양 PC가 필요한가요?**

A: *Tomodachi Life*는 그래픽 요구 사항이 높은 게임이 아닙니다. 최근 몇 년 내 출시된 대부분의 최신 중급 PC나 노트북이라면 큰 문제 없이 풀 속도로 실행할 수 있습니다.
