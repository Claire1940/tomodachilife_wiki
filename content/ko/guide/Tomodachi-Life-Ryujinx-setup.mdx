| 요구 사항 | 중요한 이유 | 2026년 권장 |
|---|---|---|
| **Ryujinx Canary 빌드** | 신규 게임 호환성과 수정 사항이 가장 먼저 반영됨 | **Canary 1.3.26.269 이상** |

| **게임 덤프 (NSP/XCI)** | 라이브러리 인식을 위한 기본 게임 파일 | 본인이 합법적으로 소유한 사본에서 깨끗하게 덤프 |

- `D:\Emulation\Switch Updates DLC\`

가능하면 지나치게 깊은 하위 폴더 구조와 경로 이름의 특수문자는 피하세요.

## Tomodachi Life Ryujinx 설정 단계별 가이드 (핵심 설치)

아래는 신뢰할 수 있는 **Tomodachi Life Ryujinx 설정**의 핵심 순서입니다. 단계를 건너뛰지 마세요.

1. **Ryujinx Canary를 다운로드합니다(해당 시 Windows x64).**  
   패키지를 압축 해제한 뒤 `publish` 폴더를 임시/다운로드 폴더가 아닌 영구 위치에 두세요.

2. **Ryujinx를 한 번 실행합니다.**  
   첫 실행에서 키/펌웨어 누락 경고가 뜨는 경우가 많습니다. 정상입니다.

3. **Ryujinx 데이터 폴더를 엽니다.**  
   앱 메뉴에서 Ryujinx 폴더를 연 뒤 `system`으로 이동하세요.

4. **키 파일을 추가합니다.**  
   `prod.keys`와 `title.keys`를 `system` 안에 넣으세요.

5. **Ryujinx를 재시작합니다.**  
   앱을 다시 열어 키 검사가 다시 실행되게 하세요.

6. **ZIP에서 펌웨어를 설치합니다.**  
   `Actions > Install Firmware > Install from ZIP`으로 이동해 펌웨어 패키지를 선택하고 확인하세요.

7. **게임 디렉터리를 설정합니다.**  
   `Options > Settings > Game Directories`에서 게임 폴더를 추가하세요.  
   업데이트/DLC를 사용한다면 autoload 아래에 해당 폴더도 추가하세요.

8. **설정을 적용하고 게임이 라이브러리에 표시되는지 확인합니다.**  
   타이틀이 보이지 않으면 파일 형식과 폴더 경로를 다시 확인하세요.

| 설치 단계 | 기대 결과 | 실패 시 |
|---|---|---|
| 키 추가 | 재시작 시 키 경고가 사라짐 | 파일명과 `system` 위치 재확인 |
| 펌웨어 설치 | 앱에 펌웨어 버전이 표시됨 | 유효한 ZIP 사용 후 재설치 |
| 게임 디렉터리 추가 | 목록에 게임 표시 | 선택한 폴더에 NSP/XCI가 있는지 확인 |
| 첫 실행 | 시작 시 셰이더 컴파일로 인한 끊김 | 정상이며 시간이 지나며 개선됨 |

> **Tip:** 펌웨어 설치 후 깔끔하게 재부팅하면 드물게 발생하는 1회성 초기화 오류가 해결될 수 있습니다.

## Tomodachi Life 권장 성능 설정

기본 설치가 끝났다면 설정은 한 번만 튜닝하세요. 아래는 2026년 **Tomodachi Life Ryujinx 설정** 사용자 대부분에게 실용적인 기본값입니다.

### 그래픽 우선순위

- 먼저 **Vulkan**을 우선 사용하세요(대개 성능이 더 좋음).
- 시스템에서 Vulkan 그래픽 깨짐이 있으면 **OpenGL**을 테스트하세요.
- **Preferred GPU**가 내장 그래픽이 아닌 외장 GPU(NVIDIA/AMD)로 설정되어 있는지 확인하세요.
- GPU 여유가 있을 때만 **2x 해상도**로 시작하세요.

| 설정 그룹 | 안전한 기본값 | 고화질 옵션 | 저사양 옵션 |
|---|---|---|---|
| **백엔드** | **Vulkan** | Vulkan + 더 높은 배율 | OpenGL |
| **해상도 배율** | **1x (네이티브)** | 2x | 1x |
| **안티 앨리어싱** | **SMAA** | SMAA Ultra | 끔 |
| **비등방성 필터링** | **8x** | 16x | 2x 또는 끔 |
| **그래픽 백엔드 멀티스레딩** | **Auto/On** | On | Auto |

### CPU 및 시스템 튜닝

- **PPTC는 켜둔 상태**를 유지하세요.
- 저사양 노트북이라면 **Low Power PPTC**를 시도하세요.
- 문제 해결 중이 아니라면 실험적 토글은 보수적으로 유지하세요.
- 고해상도 텍스처 모드를 특별히 쓰는 경우가 아니라면 DRAM은 기본값을 사용하세요.

### 입력 및 독 모드

- 먼저 컨트롤러를 매핑하고, 그다음 프로필로 저장하세요.
- Pro Controller 모드는 많은 타이틀에서 잘 작동합니다.
- 성능이 떨어지면 **휴대 모드**를 테스트하세요(독 모드 비활성화).

> **Tip:** 초반 실행에서 셰이더 캐시 끊김은 정상입니다. 최종 부드러움을 판단하기 전에 어느 정도 플레이해 보세요.

## 문제 해결: Tomodachi Life Ryujinx 설정에서 흔한 문제

대부분의 설정 실패는 작은 불일치에서 발생합니다. 아래 표로 빠르게 원인을 분리해보세요.

| 증상 | 가능성 높은 원인 | 빠른 해결 |
|---|---|---|
| **“Keys not found”** | 키 파일이 잘못된 폴더에 있거나 형식이 잘못됨 | `prod.keys`와 `title.keys`를 `system`에 넣고 재시작 |
| **펌웨어가 없음으로 표시됨** | 펌웨어를 설치하지 않았거나 설치 실패 | Actions 메뉴에서 ZIP으로 펌웨어 재설치 |
| **라이브러리에 게임이 없음** | 잘못된 디렉터리를 선택함 | NSP/XCI가 들어 있는 폴더를 직접 다시 추가 |
| **부팅 시 충돌** | Canary가 아닌 빌드 또는 잘못된 업데이트 적용 순서 | Canary로 업데이트하고 기본 게임부터 테스트 |
| **심한 끊김** | 셰이더 캐시 생성 중 | 계속 플레이하고 해상도를 일시적으로 낮춤 |
| **좋은 PC인데 FPS가 낮음** | 잘못된 GPU가 선택됨 | Ryujinx + OS GPU 설정에서 외장 GPU 수동 지정 |
| **컨트롤러가 인식되지 않음** | 실행 전 장치 초기화가 안 됨 | 먼저 연결한 뒤 Input 설정에서 새로고침 |

### 안정성 체크리스트 (이 순서대로 진행)

1. 에뮬레이터 브랜치/버전을 확인합니다.
2. 키가 정상적으로 로드되는지 확인합니다.
3. 펌웨어 설치 여부를 확인합니다.
4. 모드 없이 기본 게임을 실행합니다.
5. 업데이트를 추가합니다.
6. DLC를 추가합니다.
7. 마지막에 성능 튜닝을 적용합니다.

이 단계별 방식은 **Tomodachi Life Ryujinx 설정**을 검증하고, 여러 변수를 동시에 추적하는 상황을 피하는 가장 깔끔한 방법입니다.

## FAQ

**Q: 2026년 Tomodachi Life Ryujinx 설정에서 가장 중요한 부분은 무엇인가요?**

A: 최신 Ryujinx Canary 빌드, 올바른 `system` 폴더의 유효한 키, 그리고 정상 설치된 펌웨어를 사용하세요. 이 세 가지가 그래픽 튜닝 이전에 대부분의 실행 문제를 해결합니다.

**Q: Tomodachi Life Ryujinx 설정에서 Vulkan과 OpenGL 중 무엇을 써야 하나요?**

A: 성능을 위해 Vulkan부터 시작하세요. 특정 하드웨어/드라이버 조합에서 렌더링 깨짐이나 비정상적인 불안정성이 보이면 OpenGL을 테스트하고 여러 세션에서 안정성을 비교하세요.

**Q: Tomodachi Life Ryujinx 설정을 올바르게 했는데도 처음에 게임이 왜 끊기나요?**

A: 초반 끊김은 보통 셰이더 컴파일 때문입니다. 플레이 중 셰이더 캐시가 쌓이면 프레임 페이싱이 개선되는 경향이 있습니다. 해상도를 일시적으로 낮추면 초기 실행이 더 부드러워질 수 있습니다.

**Q: 업데이트와 DLC를 추가한 뒤 게임이 충돌합니다. 어떻게 해야 하나요?**

A: 먼저 기본 게임 실행 상태로 되돌리세요. 그다음 업데이트만 추가해 테스트하고, 마지막에 DLC를 추가하세요. 이 단계별 접근으로 충돌 원인이 업데이트 버전 불일치인지, DLC 충돌인지, 혹은 파일 자체 문제인지 식별할 수 있습니다.
