<Section icon={Shield} title="Troubleshooting Common Glitches and Fixes">

Players running the game on an emulator may encounter specific visual or audio bugs that did not exist on the original hardware. The most common issue is the missing Mii face glitch, where characters appear with blank or corrupted facial textures.

This issue occurs because the emulator lacks access to the proprietary system fonts used by the original operating system to render facial features. Resolving this requires importing the shared system files into the emulator directory.

| Issue Description | Primary Cause | Recommended Solution |
|-------------------|---------------|----------------------|
| Blank Mii Faces | Missing shared system fonts | Import system font files to the user folder |
| Black Screen on Boot | Encrypted game file | Decrypt the game file using external tools |
| Audio Crackling | Audio buffer desync | Switch audio backend to cubeb or SDL2 |
| Save Data Corrupt | Improper emulator shutdown | Always save in-game and wait for writing to finish |
| Low Frame Rates | Excessive resolution scaling | Lower internal resolution to 2x or 3x |

<Callout type="success" title="Mii Face Resolution">
After copying the shared font files into the system directory, restart the emulator. The changes will apply immediately, restoring all facial features to existing and new Miis.
</Callout>

If performance drops persist during gameplay, disabling hardware shaders or clearing the shader cache can resolve stuttering caused by compiling new visual assets on the fly.
